Historia y Filosofía de Linux

Orígenes y Evolución

Linux es un sistema operativo de tipo Unix que fue creado inicialmente como un proyecto personal por Linus Torvalds en 1991 mientras era estudiante en la Universidad de Helsinki. Su objetivo era crear un sistema operativo libre y de código abierto que pudiera ser utilizado y modificado por cualquier persona.

El núcleo (kernel) de Linux, junto con las herramientas del proyecto GNU, conformaron lo que hoy conocemos como sistema operativo GNU/Linux, una alternativa libre a los sistemas Unix propietarios de la época.

1991

Creación del Kernel Linux

Linus Torvalds anuncia la primera versión del kernel Linux.

1992-1993

Primeras Distribuciones

Aparecen las primeras distribuciones como SLS y Slackware.

1994-1995

Consolidación

Surgen Red Hat y Debian, distribuciones que aún existen hoy día.

2000s

Popularización

Linux se expande al ámbito empresarial y aparecen distribuciones más accesibles como Ubuntu.

Filosofía del Software Libre

Linux se fundamenta en los principios del software libre y el código abierto (FOSS - Free and Open Source Software), estableciendo cuatro libertades fundamentales:

  • 1 La libertad de usar el programa con cualquier propósito
  • 2 La libertad de estudiar cómo funciona el programa y modificarlo
  • 3 La libertad de distribuir copias del programa
  • 4 La libertad de mejorar el programa y hacer públicas esas mejoras

¿Sabías que?

El término "Linux" técnicamente se refiere solo al kernel (núcleo) del sistema operativo. El sistema completo es correctamente llamado GNU/Linux, ya que incluye numerosas herramientas del proyecto GNU.

Instalación y Configuración Inicial

Instalar Linux puede parecer intimidante al principio, pero con las distribuciones modernas el proceso se ha simplificado considerablemente. En esta sección aprenderemos los diferentes métodos de instalación y la configuración inicial del sistema.

Medios de Instalación

USB

El método más común actualmente. Requiere una memoria USB de al menos 4GB y software para crear unidades USB bootables.

DVD

Método tradicional utilizando discos ópticos. Menos común hoy en día pero aún funcional en equipos antiguos.

Red (PXE)

Instalación a través de la red, útil en entornos empresariales para despliegues múltiples.

Máquina Virtual

Ideal para principiantes, permite instalar Linux dentro de otro sistema operativo sin modificar el disco duro.

Instalación paso a paso

El proceso de instalación varía según la distribución, pero generalmente incluye estos pasos:

1

Preparación

Descargar la ISO de la distribución y crear medio de instalación (USB/DVD)

2

Arranque

Configurar la BIOS/UEFI para arrancar desde el medio de instalación

3

Particionado

Preparar el disco duro (partición manual o automática)

4

Instalación

Copiar archivos del sistema y configuración básica

5

Configuración

Zona horaria, teclado, usuario, red, etc.

Configuración Inicial

Sistema

  • Configuración regional

Usuarios

Actualización y Gestión de Paquetes

Sistemas de Gestión de Paquetes

Linux utiliza gestores de paquetes para instalar, actualizar y eliminar software. Los principales son:

APT (Debian/Ubuntu)

sudo apt update
sudo apt install paquete
sudo apt remove paquete

DNF/YUM (Fedora/RHEL)

sudo dnf update
sudo dnf install paquete
sudo dnf remove paquete

Pacman (Arch Linux)

sudo pacman -Syu
sudo pacman -S paquete
sudo pacman -R paquete

Zypper (openSUSE)

sudo zypper refresh
sudo zypper install paquete
sudo zypper remove paquete

Buenas Prácticas

Actualización Regular

Mantén tu sistema actualizado para recibir parches de seguridad y mejoras.

Repositorios Oficiales

Prioriza la instalación desde los repositorios oficiales de tu distribución.

Consulta la Documentación

Antes de instalar paquetes de fuentes externas, lee la documentación oficial.

Limpieza Periódica

Elimina paquetes no utilizados y limpia la caché del gestor de paquetes.

Precaución

Manipular los paquetes del sistema puede afectar la estabilidad. Siempre realiza copias de seguridad antes de modificaciones importantes.

Comandos y Herramientas de la Línea de Comandos

Uso de la Terminal

La terminal (también conocida como consola o línea de comandos) es una interfaz de texto que permite controlar el sistema mediante comandos. Es una herramienta poderosa y esencial en Linux.

Terminal

usuario@linux:~$ _

Ventajas de la Terminal

  • Control preciso sobre el sistema
  • Automatización de tareas repetitivas
  • Menor consumo de recursos que interfaces gráficas
  • Acceso remoto eficiente (SSH)

Manipulación de Archivos y Directorios

Navegación

# Ir al directorio home
cd ~

# Directorio actual
pwd

# Listar archivos (incluyendo ocultos)
ls -la

# Ir al directorio padre
cd ..

# Ir a directorio específico
cd /etc/

Creación y Eliminación

# Crear directorio
mkdir nuevo_directorio

# Crear archivo vacío
touch archivo.txt

# Eliminar archivo
rm archivo.txt

# Eliminar directorio vacío
rmdir directorio

# Eliminar directorio con contenido
rm -r directorio

Visualización y Edición

# Ver contenido
cat archivo.txt

# Editar con nano
nano archivo.txt

# Editar con vim
vim archivo.txt

# Buscar texto en archivos
grep "texto" archivo.txt

Precaución

El comando rm elimina archivos permanentemente. No hay papelera de reciclaje en la línea de comandos. Ten especial cuidado con rm -rf que elimina de forma recursiva y forzada.
