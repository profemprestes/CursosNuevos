Procesos y Gestión de Recursos

Conceptos Básicos de Procesos

En Linux, un proceso es una instancia de un programa en ejecución. Cada proceso tiene su propio espacio de memoria, variables y recursos asignados por el sistema operativo.

Características de los Procesos

  • 1 Cada proceso tiene un identificador único (PID)
  • 2 Los procesos pueden crear otros procesos (procesos hijo)
  • 3 Pueden ejecutarse en primer plano o en segundo plano
  • 4 Tienen diferentes estados: ejecución, espera, detenido, etc.

Administración de Usuarios y Grupos

Linux es un sistema multiusuario que permite a múltiples usuarios acceder simultáneamente al sistema. La administración adecuada de usuarios y grupos es fundamental para mantener la seguridad y organización del sistema.

Sistema de Usuarios en Linux

Tipos de Usuarios

Root (Superusuario)

UID 0, privilegios completos sobre el sistema

Usuarios del Sistema

UID 1-999, servicios y daemons del sistema

Usuarios Regulares

UID 1000+, usuarios normales con privilegios limitados

Archivos del Sistema Relacionados con Usuarios

/etc/passwd
Base de datos principal de usuarios
nombre:x:UID:GID:comentario:home:shell
/etc/shadow
Almacena contraseñas cifradas
/etc/group
Base de datos de grupos
/etc/gshadow
Contraseñas cifradas de grupos

Herramientas de Administración

Herramientas de Línea de Comandos

  • useradd - Crear usuarios
  • usermod - Modificar usuarios
  • userdel - Eliminar usuarios
  • passwd - Gestionar contraseñas
  • groupadd - Crear grupos
  • groupmod - Modificar grupos

Herramientas Gráficas

  • system-config-users - Red Hat/Fedora
  • users-admin - Ubuntu/GNOME
  • YaST User Management - openSUSE

Ejemplo de Información de Usuario

Terminal
$ id usuario1
uid=1000(usuario1) gid=1000(usuario1) grupos=1000(usuario1),4(adm),24(cdrom),27(sudo)

$ grep usuario1 /etc/passwd
usuario1:x:1000:1000:Usuario Ejemplo:/home/usuario1:/bin/bash

Gestión de Cuentas de Usuario

La creación de usuarios en Linux implica varios pasos automáticos, incluyendo la asignación de UID/GID y la creación del directorio home.

Creación de Usuarios

La creación de usuarios en Linux se puede realizar mediante el comando useradd.

1

Crear Usuario Básico

sudo useradd nombreusuario
2

Crear Usuario con Opciones

sudo useradd -m -d /home/usuario -s /bin/bash -c "Nombre Completo" usuario
3

Establecer Contraseña

sudo passwd nombreusuario
4

Verificar Usuario

id nombreusuario
grep nombreusuario /etc/passwd

Opciones Comunes de useradd

Opción Descripción
-m Crear directorio home
-d Especificar ruta del directorio home
-s Especificar shell
-c Comentario (generalmente nombre completo)
-G Grupos adicionales

Eliminación y Modificación de Usuarios

Eliminación de Usuarios

Para eliminar un usuario en Linux, se utiliza el comando userdel.

# Eliminar usuario manteniendo su directorio home
sudo userdel nombreusuario

# Eliminar usuario y su directorio home
sudo userdel -r nombreusuario

# Forzar eliminación incluso si el usuario tiene procesos en ejecución
sudo userdel -f nombreusuario
Precaución

La eliminación de usuarios es permanente. Considera hacer una copia de seguridad de los datos importantes del directorio home antes de usar la opción -r.

Modificación de Usuarios

El comando usermod permite modificar diversos atributos de las cuentas de usuario existentes.

# Cambiar shell del usuario
sudo usermod -s /bin/zsh nombreusuario

# Añadir usuario a grupos adicionales
sudo usermod -aG grupo1,grupo2 nombreusuario

# Cambiar directorio home
sudo usermod -d /nuevo/home/ruta nombreusuario

# Cambiar nombre de usuario
sudo usermod -l nuevonombre antiguonombre

# Bloquear cuenta de usuario
sudo usermod -L nombreusuario

# Desbloquear cuenta de usuario
sudo usermod -U nombreusuario

Ejemplo Práctico

1

Crear un usuario para un nuevo desarrollador:

sudo useradd -m -s /bin/bash -c "Ana Desarrolladora" -G developers,sudo anadev
2

Establecer una contraseña temporal:

sudo passwd anadev
3

Obligar a cambiar la contraseña en el primer inicio de sesión:

sudo passwd -e anadev

Gestión de Contraseñas

Administración de Contraseñas

La gestión adecuada de contraseñas es fundamental para mantener la seguridad del sistema Linux. Las contraseñas se almacenan de forma cifrada en el archivo /etc/shadow.

Comandos Principales

passwd

Permite a los usuarios cambiar sus contraseñas y a los administradores gestionar contraseñas de otros usuarios.

# Usuario cambia su propia contraseña
passwd

# Administrador cambia la contraseña de otro usuario
sudo passwd nombreusuario

# Bloquear contraseña de usuario
sudo passwd -l nombreusuario

# Desbloquear contraseña de usuario
sudo passwd -u nombreusuario

# Forzar cambio de contraseña en próximo inicio de sesión
sudo passwd -e nombreusuario

Archivo /etc/shadow

Cada línea en /etc/shadow contiene los siguientes campos separados por dos puntos:

username:encrypted_password:lastchange:min:max:warn:inactive:expire:reserved
  • username: Nombre del usuario
  • encrypted_password: Contraseña cifrada (o * si está bloqueada)
  • lastchange: Días desde 1/1/1970 que se cambió la contraseña
  • min: Días mínimos entre cambios de contraseña
  • max: Días máximos de validez de la contraseña
  • warn: Días antes de max para advertir al usuario
  • inactive: Días después de expirar para desactivar la cuenta
  • expire: Fecha de caducidad de la cuenta (días desde 1/1/1970)

Política de Contraseñas

Configuración de Políticas

Las políticas de contraseñas en Linux pueden gestionarse mediante varios mecanismos:

Módulo PAM (pam_pwquality)

Configurable en /etc/security/pwquality.conf o /etc/pam.d/common-password

Herramienta chage

Permite configurar parámetros de caducidad de contraseñas

Uso de chage

# Ver configuración actual de un usuario
sudo chage -l nombreusuario

# Establecer fecha de expiración de contraseña 
sudo chage -E 2023-12-31 nombreusuario

# Configurar días máximos de validez
sudo chage -M 90 nombreusuario

# Configurar advertencia previa a expiración
sudo chage -W 7 nombreusuario

# Forzar cambio de contraseña en próximo inicio de sesión
sudo chage -d 0 nombreusuario

Buenas Prácticas de Seguridad

Longitud Mínima

Establecer contraseñas de al menos 12 caracteres

Complejidad

Combinar mayúsculas, minúsculas, números y caracteres especiales

Caducidad

Forzar cambios regulares (60-90 días)

Gestión de Grupos

Conceptos de Grupos en Linux

Los grupos en Linux permiten organizar usuarios y asignar permisos colectivamente, facilitando la administración del sistema.

Tipos de Grupos

Grupo Primario

Cada usuario pertenece a un grupo primario, generalmente con el mismo nombre del usuario.

Grupos Secundarios

Un usuario puede pertenecer a múltiples grupos secundarios para obtener permisos adicionales.

Grupos del Sistema

GID entre 0-999, usados para procesos del sistema (audio, video, sudo, etc.).

Archivo /etc/group

La información de grupos se almacena en /etc/group con el siguiente formato:

nombregrupo:x:GID:lista,de,usuarios
  • nombregrupo: Nombre del grupo
  • x: Marcador de contraseña (actual en /etc/gshadow)
  • GID: ID numérico del grupo
  • lista,de,usuarios: Usuarios miembros del grupo (secundario)

Administración de Grupos

Creación y Eliminación de Grupos

# Crear un nuevo grupo
sudo groupadd nombregrupo

# Crear grupo con GID específico
sudo groupadd -g 1500 nombregrupo

# Eliminar un grupo
sudo groupdel nombregrupo

Modificación de Grupos

# Cambiar nombre del grupo
sudo groupmod -n nuevonombre viejonombre

# Cambiar GID de un grupo
sudo groupmod -g 1600 nombregrupo

Gestión de Miembros

# Añadir usuario a grupo
sudo usermod -aG nombregrupo usuario

# Añadir usuario a múltiples grupos
sudo usermod -aG grupo1,grupo2,grupo3 usuario

# Establecer grupo primario de un usuario
sudo usermod -g nombregrupo usuario

# Añadir usuario directamente al grupo (alternativa)
sudo gpasswd -a usuario nombregrupo

# Eliminar usuario de un grupo
sudo gpasswd -d usuario nombregrupo

# Ver grupos de un usuario
groups usuario

Ejemplo Práctico

1

Crear un grupo para el departamento de desarrollo:

sudo groupadd developers
2

Añadir usuarios al grupo:

sudo usermod -aG developers usuario1 sudo usermod -aG developers usuario2
3

Crear carpeta compartida para el grupo:

sudo mkdir /var/shared/developers sudo chown root:developers /var/shared/developers sudo chmod 2775 /var/shared/developers
